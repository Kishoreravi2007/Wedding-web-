# Wedding Web Application - Complete Service Startup Plan

## Project Overview
Full-stack wedding photo gallery with face recognition, built with:
- **Backend**: Node.js/Express API (Port 5001)
- **Frontend**: React/Vite (Port 3000-3002)
- **Database**: Supabase
- **Face Recognition**: Python services (DeepFace/InsightFace)
- **Features**: Photo upload, face detection, photographer portal

## Services to Start

### 1. Backend Services
- [ ] Main Express API server (Port 5001)
- [ ] WebSocket server for real-time updates
- [ ] Supabase integration
- [ ] Face recognition API endpoints

### 2. Frontend Services  
- [ ] React development server (Vite)
- [ ] Hot reload for development
- [ ] Multi-language support (English/Malayalam)

### 3. Face Detection Services
- [ ] DeepFace API service
- [ ] InsightFace recognition service
- [ ] Face descriptor processing

### 4. Additional Services
- [ ] Photo upload handling
- [ ] Live sync services
- [ ] Contact form processing
- [ ] Analytics tracking

## Startup Sequence
1. Check dependencies and environment
2. Start backend server (Port 5001)
3. Start frontend server (Port 3000)
4. Verify all services are running
5. Test API connectivity
6. Open web interface

## Expected URLs
- Frontend: http://localhost:3000
- Backend API: http://localhost:5001
- Health Check: http://localhost:5001/api/health
- API Info: http://localhost:5001/api

## Success Criteria
- [ ] Backend responding on port 5001
- [ ] Frontend accessible on port 3000
- [ ] No critical errors in logs
- [ ] Database connectivity confirmed
- [ ] Face recognition endpoints available
